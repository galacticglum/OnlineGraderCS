Contest Rules

1. Implementation

All input is from standard input and all output is from standard output. Programs submitted to the grader should not read or write files.

If you are using Java, the class containing your main method MUST be called "Main"!

Outputs must exactly match the format specified in the problem. Prompts or additional output will render the output invalid.


2. Submission

The grader will accept up to 50 submissions per problem. Users may submit at most once per minute to any specific problem.


3. Resources, References, and Tools

During the contest, contestants may use the internet for problem submissions and for reference consultation. Official documentation sources are allowed, examples include:

  • Python 2 Documentation
  • Python 3 Documentation
